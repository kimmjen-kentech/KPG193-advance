# KPG-193 실시간 시뮬레이션 연구 — 비전공자를 위한 상세 설명서

> 이 문서는 전기공학 지식 없이도 읽을 수 있도록 작성되었습니다.  
> 원본: 2026-05-19 발표 자료 "전국전력망 축약계통(KPG-193)의 실시간 시뮬레이션을 위한 Multi-core Co-simulation 구현"

---

## 1. 이 연구는 무엇을 하려는 건가요?

한 줄 요약: **한국 전력망 전체를 컴퓨터로 실시간 시뮬레이션하기 위한 방법을 개발했습니다.**

조금 더 풀어보면:

- 한국에는 수백 개의 발전소, 수천 km의 송전선이 얽혀 있습니다.
- 이 전력망에서 갑자기 발전소 하나가 꺼지면? 어떤 선로에 과부하가 걸릴까? 주파수가 얼마나 흔들릴까?
- 이걸 실제 전력망에서 직접 실험할 수는 없습니다 (정전 대란이 납니다).
- 그래서 컴퓨터로 **똑같이 흉내**를 내봅니다 — 이게 시뮬레이션입니다.
- 문제는 한국 전력망이 너무 크고 복잡해서, 기존 방식으로는 컴퓨터가 감당을 못 한다는 것입니다.
- 이 연구는 **여러 컴퓨터 코어(계산 단위)에 나눠서 동시에 계산**하는 방법(Co-simulation)을 개발했습니다.

## 2. KPG-193이란? — 한국 전력망의 "축약 모형"

### 왜 실제 데이터를 못 쓰나요?

한국전력(한전)이 관리하는 실제 전력망 데이터는 **보안상 비공개**입니다. 어떤 발전소가 어디에 있고, 선로가 어떻게 연결되는지 그 상세 정보가 외부에 공개되면 사이버 공격 등의 위험이 있습니다.

### 그래서 만든 것이 KPG-193

공개된 데이터(기상청 기온·풍속·일사량, 한전 지역 사무소 위치, 통계청 전력 사용량 등)를 조합해서 **실제 한국 전력망과 비슷하게 동작하는 모형**을 만들었습니다.

| 항목 | 숫자 | 쉬운 설명 |
|------|------|-----------|
| 모선(Bus) | 193개 | 발전소·변전소 같은 전기 "교차점" 수 |
| 송전선(Branch) | 407개 | 교차점들을 잇는 "전기 도로" 수 |
| 발전기 | 122개 | 전기를 만드는 장치 수 |

**특징:**
- 수도권(서울·경기)에 소비가 집중
- 서남부(전남·충남)에 태양광·풍력 집중
- 해안가(동·서해)에 화력·원자력 발전단지

이 모형은 논문으로도 발표되어 있습니다:  
Song & Kim, "KPG 193: A Synthetic Korean Power Grid Test System for Decarbonization Studies," arXiv:2411.14756, 2024

## 3. 실시간 시뮬레이션이란? — 컴퓨터가 현실을 따라잡아야 한다

### 보통 시뮬레이션 vs 실시간 시뮬레이션

- **보통 시뮬레이션**: 현실에서 1시간 일어날 일을 컴퓨터가 10분에 계산해도 됩니다. 빨리 끝나면 끝날수록 좋습니다.
- **실시간 시뮬레이션**: 현실에서 1초가 지나는 동안 컴퓨터도 정확히 1초치 계산을 끝내야 합니다. 한 발짝도 뒤처지면 안 됩니다.

### 왜 실시간이어야 하나요?

**HIL(Hardware-in-the-Loop)** 때문입니다. HIL이란 실제 하드웨어 장비(예: 제어기, 보호 계전기)를 가상의 시뮬레이션과 **실시간으로 연결**해서 테스트하는 방식입니다.

예를 들어:
- 새로 개발한 **전력망 보호 장치**를 실제 전력망에 바로 붙이면 위험합니다.
- 대신 컴퓨터가 전력망을 실시간으로 흉내 내면, 이 장치가 "진짜 전력망"인 줄 알고 반응합니다.
- 이렇게 안전하게 테스트할 수 있습니다.

### Over-run: 시간 안에 계산을 못 끝내면?

실시간 시뮬레이션에서 각 계산 주기를 **Time-step**이라고 합니다. 예를 들어 50마이크로초(0.00005초)마다 계통 상태를 갱신해야 한다면, 그 시간 안에 계산을 끝내야 합니다.

계산이 너무 복잡해서 시간을 초과하면 **Over-run**이 발생합니다. 이러면 시뮬레이션이 현실 시간을 따라가지 못하고 무너집니다.

> **비유**: 축구 중계 방송에서 카메라맨이 공의 움직임을 1초 뒤에야 찍는다면, 그건 중계가 아닙니다. 실시간 시뮬레이션도 마찬가지 — 현실과 동시에 움직여야 합니다.

## 4. EMT vs RMS — 두 가지 계산 방식의 차이

전력망을 시뮬레이션하는 방식은 크게 두 가지입니다.

### EMT (Electromagnetic Transients) — 전자기 과도 해석

**전압과 전류의 순간순간 파형(물결 모양)을 그대로 계산**합니다.

> **비유**: 고속 카메라로 물방울이 튀는 순간을 초당 수만 프레임으로 찍는 것. 아주 세밀하지만, 데이터가 엄청나게 많고 처리가 느립니다.

- Time-step: **2~50마이크로초** (1초의 5만분의 1 수준)
- 장점: 태양광 인버터, HVDC(초고압 직류 송전) 같은 빠른 전자기 현상을 정확히 볼 수 있음
- 단점: 매우 느림 → 대규모 계통에 적용하면 Over-run 발생
- 적합 규모: **100개 모선 이하** 소규모 계통

### RMS (Root Mean Square) — 실효값 해석

**전압과 전류의 "평균적인 크기와 위상"만 계산**합니다. 파형의 세세한 모양은 무시하고, 핵심 정보만 추립니다.

> **비유**: 스마트폰 카메라로 초당 30프레임 촬영. 빠른 움직임의 세부는 못 잡지만, 일상 촬영에는 충분하고 훨씬 가볍습니다.

- Time-step: **50~200밀리초** (EMT보다 수천 배 느린 간격)
- 장점: 빠름 → 대규모 계통, 장시간 시뮬레이션 가능
- 단점: 순간적인 전자기 현상은 못 봄
- 적합 규모: **1000개 모선 이상** 대규모 계통

### 한눈에 비교

| 구분 | EMT | RMS |
|------|-----|-----|
| 무엇을 계산하나 | 전압·전류 순간 파형 | 전압·전류 평균 크기·위상 |
| 계산 속도 | 매우 느림 | 빠름 |
| 정밀도 | 매우 높음 | 중간 |
| 적합 규모 | 소규모 (100모선 이하) | 대규모 (1000모선 이상) |
| Time-step | 2~50 μs | 50~200 ms |
| 주요 도구 | PSCAD, EMTP | PSS/E, TSAT |

## 5. 각 방식으로 어떻게 계통을 모델링하나요?

"모델링"이란 실제 장비를 컴퓨터 수식으로 표현하는 것입니다.

### 5-1. 동기 발전기 (화력·원자력·수력 발전소)

**동기 발전기**란 발전소에 있는 거대한 회전 기계입니다. 자석이 빠르게 돌면서 전기를 만듭니다.

이 발전기를 모델링할 때 중요한 제어 장치 세 가지:

- **AVR (자동전압조정기)**: 발전기 출력 전압이 흔들리면 자동으로 보정
- **Governor (거버너)**: 발전기 회전 속도(= 주파수)가 흔들리면 연료 공급을 조절
- **PSS (전력계통안정화장치)**: 계통 전체가 흔들릴 때 발전기가 이를 안정시키도록 보조

**EMT 방식**: 발전기 내부 코일의 자기장 변화까지 매우 상세하게 계산  
**RMS 방식**: GENROU라는 표준 수식 모델 사용 — 핵심 거동만 수식으로 표현하여 계산량 줄임

**검증 결과**: 부하(전력 소비)가 갑자기 바뀔 때 주파수가 어떻게 응답하는지 시뮬레이션 결과가 이론값과 일치함을 확인.

---

### 5-2. 송전선로와 변압기

**송전선로**: 발전소에서 만든 전기를 멀리 보내는 고압 전선입니다.  
**변압기**: 전압을 바꿔주는 장치 (예: 154kV → 22.9kV로 낮춰서 가정에 공급)

모델에서는 **파이(π)형 집중정수 모델**을 사용합니다.

> **비유**: 수도관에서 물이 흐를 때 관의 저항, 탄성, 누수를 각각 숫자로 나타내는 것처럼, 전선도 저항(R), 인덕턴스(L), 커패시턴스(C) 세 가지 값으로 표현합니다.

**검증 결과**: 여러 전압 레벨이 섞인 계통에서 각 모선의 전압과 위상각이 이론값과 동일함을 확인.

---

### 5-3. IBR — 태양광·풍력 같은 인버터 발전원

**IBR(Inverter-Based Resources)**: 태양광 패널, 풍력 터빈, 배터리 저장장치처럼 **인버터(전력변환장치)를 통해 전기를 만드는 자원**입니다.

전통적인 발전기(회전 기계)와 달리:
- 물리적으로 회전하지 않아 관성이 없습니다.
- 전기를 직접 "전류"로 내보내는 방식으로 동작합니다.

**Grid-Following (계통 추종) 방식**으로 동작:
- 기존 전력망의 주파수·전압을 감지하고 거기에 맞춰 출력합니다.
- 지령값에 따라 발전(+)도 하고 부하(-)처럼 전력을 소비하기도 합니다.

재생에너지가 늘어날수록 이 IBR의 비중이 커지기 때문에, 정확한 모델링이 중요합니다.

## 6. AutoGrid — 계통을 자동으로 그려주는 도구

### 문제: 193개 모선을 손으로 연결하면?

발전기 122개, 선로 407개를 Simulink(컴퓨터 시뮬레이션 도구)에서 하나하나 배치하고 연결하는 것은 며칠이 걸리고, 실수도 많이 납니다.

### 해결책: AutoGrid

**MATPOWER**라는 전력망 표준 데이터 형식(엑셀 비슷한 데이터 파일)을 입력하면:
1. 모선을 위치에 맞게 자동 배치
2. 각 모선 타입(Slack/PV/PQ)에 맞는 구성요소 자동 연결
3. 송전선로 자동 배선

결과:
- Human Error(사람 실수) **0%**
- 전압·전력 오차 **0.1% 이내**
- 작업 시간 대폭 단축

> **비유**: CAD 프로그램이 설계도 치수를 보고 자동으로 회로를 그려주는 것과 같습니다.

## 7. 왜 여러 컴퓨터를 나눠서 계산해야 하나요?

### 단일 코어의 한계

하나의 CPU 코어로 EMT 시뮬레이션을 할 때, **실시간으로 처리할 수 있는 규모는 약 10개 모선 수준**입니다.

KPG-193은 193개 모선이니, 단일 코어로는 **19배** 이상 부족합니다.

6개 코어를 모두 써도 KPG-193 전체를 EMT로 실시간 처리하기에는 역부족입니다.

### 두 가지 해결 방향

**방법 1: 계통을 여러 조각으로 쪼개서 각 코어에 배분**  
→ Core 1이 A구역, Core 2가 B구역, Core 3이 C구역... 동시에 계산

**방법 2: 관심 있는 부분만 EMT, 나머지는 RMS**  
→ 상세히 봐야 할 구역(예: 태양광 인버터가 많은 구역)은 EMT  
→ 나머지 넓은 전력망은 RMS로 빠르게 계산

이 두 방법을 결합한 것이 **Multi-core Co-simulation**입니다.

## 8. Co-simulation의 세 가지 방식

"Co-simulation"이란 여러 시뮬레이션이 **협력해서 하나의 큰 시스템을 계산**하는 것입니다.

분할된 구역끼리 데이터를 교환하는 방식으로 **ITM(이상적 변압기 모델)** 인터페이스를 사용합니다.

> **비유**: 두 나라가 국경에서 서로 물건을 주고받는 것. 한쪽은 "전압(전위차)"를 보내고, 다른 쪽은 "전류(흐름)"를 돌려줍니다.

---

### 8-1. EMT-EMT Co-Simulation

**두 구역 모두 EMT 방식으로 계산, 서로 순시값(파형 그대로) 교환**

- 같은 "언어"(시간 영역)를 쓰므로 번역 불필요
- 경계면에서 전압 파형 → 전류 파형을 실시간 교환

**결과**: 33개 모선 계통을 두 코어로 나눠 계산했을 때, 단일 계통 계산 결과와 버스 전압이 일치함을 확인.

---

### 8-2. RMS-RMS Co-Simulation

**두 구역 모두 RMS 방식으로 계산, 서로 페이저(크기·위상) 교환**

- 같은 "언어"(페이저 영역)를 쓰므로 번역 불필요
- 경계면에서 전압 크기·위상 → 전류 크기·위상을 실시간 교환

**결과**: IEEE 15-계통을 3개 코어로 나눠 계산, 전압·위상·선로 조류 모두 일치 확인.

---

### 8-3. RMS-EMT Co-Simulation ← 이 연구의 핵심

**한 구역은 RMS, 다른 구역은 EMT — 서로 다른 "언어"를 쓰는 상황**

여기서 문제가 생깁니다:
- EMT는 순간 파형(예: `220 × sin(2π×60×t + 30°)`)을 주고받습니다.
- RMS는 크기·위상(예: "전압 220V, 위상각 30도")을 주고받습니다.

이 둘 사이의 **번역(데이터 변환)**이 필요합니다.

**EMT → RMS 변환: 푸리에 변환(Fourier Transform)**  
- 순간 파형에서 기본 주파수(60Hz) 성분의 크기와 위상을 추출  
- 파형의 "평균적 특성"을 뽑아내는 수학 기법

**RMS → EMT 변환: 파형 복원**  
- 크기(V)와 위상(°)을 받아서 3상 순간 파형으로 복원  
- `v_a(t) = V × cos(2π×60t + φ)` 형태로 재생성

> **비유**: 영어로 말하는 사람(EMT)과 한국어로 말하는 사람(RMS)이 통역사(데이터 변환 알고리즘)를 통해 대화하는 것.

**결과**: IEEE 18-계통을 2코어(RMS + EMT)로 나눠 계산, 전압·조류 일치 확인.

## 9. 한국 전력망(KPG-193)에 실제로 적용한 결과

### 9-1. KPG-193 단독 시뮬레이션 (SPEEDGOAT)

**시나리오**: 전체 발전량 17,600MW 중 270MW 발전원이 갑자기 꺼진다.  
(270MW ≈ 전체의 1.5% — 중형 화력발전소 하나가 갑자기 탈락하는 상황)

**결과**:
- 주파수가 60Hz → **59.64Hz**까지 떨어짐 (정상은 60Hz)
- 수초 이내 자동으로 회복
- 전압 오차: 0.0000475 pu (거의 오차 없음)

> **참고**: 한국 전력망은 60Hz를 기준으로 운영됩니다. 주파수가 너무 많이 떨어지면 정전이 발생합니다.

---

### 9-2. RMS-EMT Multi-core Co-Simulation

**구성**:
- Core 1: RMS 방식 (2000마이크로초 주기) — 대부분의 KPG-193 계통
- Core 2: EMT 방식 (50마이크로초 주기) — 상세 해석이 필요한 일부 구역

**결과**:
- 전체 193개 모선의 전압 변동 패턴이 기준값과 **유사하게 일치**
- 이종(RMS+EMT) 솔버 간 데이터 동기화 알고리즘 유효성 검증
- 수치적 대수 루프(계산이 무한 반복되는 문제) 극복 → 정상 수렴

## 10. 결론 — 이 연구가 왜 중요한가요?

### 해결한 문제

| 문제 | 해결 방법 |
|------|-----------|
| 한국 전력망 데이터가 비공개 | 공개 데이터로 합성 계통(KPG-193) 구축 |
| 대규모 계통을 EMT로 실시간 해석 불가 | 멀티코어 분할 Co-simulation |
| EMT는 정밀하지만 느리고, RMS는 빠르지만 부정확 | RMS-EMT 하이브리드 Co-simulation |
| MATPOWER 데이터 → Simulink 수작업 시간 소요 | AutoGrid 자동화 도구 개발 |

### 이 연구가 가능하게 하는 것

1. **재생에너지 확대 영향 분석**: 태양광·풍력이 늘어날 때 전력망이 어떻게 반응하는지 시뮬레이션
2. **제어 장치 사전 검증**: 새로운 보호 계전기나 제어 알고리즘을 실제 전력망에 붙이기 전에 안전하게 테스트
3. **사고 시나리오 반복 재현**: 실제로는 한 번도 일어나선 안 될 대형 사고를 가상으로 반복 실험
4. **탈탄소 전환 연구**: 석탄·가스 발전을 재생에너지로 대체할 때의 안정성 검토

### 핵심 성과 정리

- KPG-193 계통의 EMT/RMS 모델을 Simulink에서 자동 생성 (AutoGrid)
- EMT-EMT, RMS-RMS, RMS-EMT 세 가지 Co-simulation 방식 모두 구현 및 검증
- KPG-193 전체를 SPEEDGOAT 실시간 시뮬레이터에서 구동 성공
- 발전원 탈락 시나리오에서 주파수 응답 및 전압 회복 확인

## 용어 정리

| 용어 | 쉬운 설명 |
|------|-----------|
| 모선 (Bus) | 전력망의 교차점. 발전소, 변전소, 부하 연결점 |
| 송전선 (Branch) | 모선과 모선을 연결하는 전선 |
| 발전기 (Generator) | 전기를 만드는 장치 |
| 부하 (Load) | 전기를 소비하는 장치·건물 |
| 주파수 (Hz) | 전기의 진동 속도. 한국은 60Hz 기준 |
| 전압 (kV) | 전기의 세기. 가정용 220V, 송전선은 154~765kV |
| 조류 (Power Flow) | 전력망에서 전력이 흐르는 방향과 양 |
| EMT | 순간 전압·전류 파형을 계산하는 정밀 시뮬레이션 |
| RMS | 전압·전류의 평균 크기·위상만 계산하는 고속 시뮬레이션 |
| Co-simulation | 여러 시뮬레이터가 협력해 하나의 큰 시스템을 계산 |
| Over-run | 시뮬레이션이 정해진 시간 안에 계산을 못 끝내는 오류 |
| IBR | 인버터 기반 발전원 (태양광, 풍력, 배터리 등) |
| MATPOWER | 전력망 데이터 표준 형식 (오픈소스) |
| SPEEDGOAT | 실시간 시뮬레이션 전용 하드웨어 플랫폼 |
| Simulink | MATLAB 기반 시스템 모델링·시뮬레이션 도구 |
| HIL | Hardware-in-the-Loop. 실제 하드웨어와 가상 시뮬레이션을 연결하는 테스트 방식 |
| ITM | Ideal Transformer Model. Co-simulation에서 두 구역을 연결하는 인터페이스 |
| HVDC | 초고압 직류 송전. 교류 대신 직류로 먼 거리 전력을 보내는 방식 |
| AVR | Automatic Voltage Regulator. 발전기 출력 전압 자동 조정 장치 |
| Governor | 발전기 회전 속도(주파수) 조정 장치 |
| PSS | Power System Stabilizer. 전력계통 진동 억제 장치 |
